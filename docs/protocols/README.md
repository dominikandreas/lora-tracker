# Protocol specifications

- `EQUINE_PROTOCOL_V1.md`: versioned frame envelope, history schema v1 and ACKs
- `EQUINE_PROTOCOL_V2.md`: history schema v2 delta timestamps
- `EQUINE_CONFIG_V1.md`: persistent tracker/gateway configuration
- `EQUINE_ONBOARDING_API_V1.md`: Wi-Fi/HTTP and BLE configuration transport
- `EQUINE_MQTT_API_V1_POINT_SCHEMA_V1.md`: original MQTT point/history schema
- `EQUINE_MQTT_API_V1_POINT_SCHEMA_V2.md`: current timestamp-aware MQTT schema

The topic API remains v1 while individual JSON message schemas can advance.
